### 구성

조회 결과 출력 중심의 화면으로, 상단의 컨텍스트·요약 영역과 하단의 목록 영역으로 구분된다. 상단에는 현재 보고 있는 구역을 나타내는 브레드크럼과 데이터 연결 상태 배지, 납기 임박 건수를 강조하는 배너, 그리고 진행 상태 필터와 총 건수가 배치된다. 하단에는 주문 ID·구역·진행 상태·우선순위·수량·계획일·납기를 보여주는 주문 테이블과 페이지네이션이 위치하며, 우선순위·계획일·납기 컬럼은 헤더의 화살표로 오름/내림차순 정렬할 수 있다.

### 처리절차

1. 구역 선택 (전체 / 구역 A·B)
2. 상태 필터 선택 (전체 / 대기 / 진행중 / 완료)
3. 컬럼 정렬 (우선순위 · 계획일 · 납기, 오름/내림 토글)
4. 페이지 이동으로 주문 탐색
5. 주문 선택 → 주문 상세 열기

### 제약사항

- 로그인 사용자 전용
- 납기 임박 = 시뮬레이션 현재일 기준 (시뮬 정지 시 계획일/서버 플래그로 폴백)
- 정렬은 우선순위·계획일·납기 컬럼만, 페이지당 8건
- API 미연결 시 데모(mock) 데이터로 표시

### 구성

목록 화면에서 주문을 선택하면 우측에서 슬라이드되어 나타나는 패널로, 헤더와 본문으로 구분된다. 헤더에는 주문 ID와 긴급 여부, 구역·진행 상태 칩, 닫기 버튼이 배치된다. 본문에는 계획일·납기·계획 수량·우선순위를 담은 주문 정보와 전체 진행률 바, 그리고 소속 유닛별 공정 파이프라인(A→B→C→D)이 표시된다. 각 유닛은 상태에 따라 완료(완료 시각), 진행중(현재 STEP·장비와 완료 예상 시각), 대기(회색)로 구분되어 표기된다.

### 처리절차

1. 목록에서 주문 선택 (패널 열림)
2. 주문 정보·전체 진행률 확인
3. 유닛별 공정 진행 상황 확인
4. 닫기

### 제약사항

- 목록에서 주문 선택 시에만 표시
- 유닛 공정 정보는 상세 조회 시점 기준
- API 미연결 시 데모(mock) 데이터로 표시
