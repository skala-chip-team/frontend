# UI 정의서: 작업자 관리

작업자 관리는 ADMIN 전용 기능이며 사이드바 운영자 섹션에서 들어간다. 전체 페이지와, 작업자를 클릭했을 때 열리는 상세 패널 두 화면으로 나눠 정의한다.

---

| 1 | 업무 Level1 | 운영 관리 |

| 4 | 화면명 | 작업자 관리 |

### 화면 레이아웃

```
┌─────────────────────────────────────────────────────────────────────┐
│ 작업자 관리                                                           │  위치 브레드크럼
├─────────────────────────────────────────────────────────────────────┤
│ 구역별 요약 (전체 선택 시)                                            │

│ │ 운영자  홍길동   │ │ 운영자  미배정   │                              │

├─────────────────────────────────────────────────────────────────────┤

│   ...                                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

### 구성

상단의 컨텍스트와 하단의 목록으로 나뉜다. 맨 위에는 현재 보고 있는 위치를 나타내는 브레드크럼이 있다. 헤더에서 전체를 선택한 경우, 구역별로 운영자와 작업자 인원을 요약한 카드를 보여준다. 그 아래에는 역할 필터(전체, 관리자, 운영자, 작업자)가 있고, 목록 테이블은 이메일, 유저 이름, 역할(색상 칩), 권한 구역, 계정 상태(활성 또는 비활성)를 표시한다. 행을 클릭하면 우측 상세 패널이 열린다.

### 처리절차

1. 구역 선택 (전체 또는 특정 구역)
2. 역할 필터 선택 (전체, 관리자, 운영자, 작업자)
3. 목록에서 작업자 선택
4. 상세 패널에서 역할과 권한 구역 확인 및 수정

### 제약사항

1. ADMIN 권한 전용 (사이드바 운영자 섹션, `/workers`)
2. 특정 구역을 선택하면 그 구역에 권한이 있는 작업자만 노출된다
3. 역할 칩 색: 운영자는 빨강 계열, 작업자는 초록 계열, 관리자는 회색
4. API 미연결 시 데모(mock) 데이터로 표시

## 2. 작업자 상세

### 화면 기본 정보

| No. | 항목명 | 내용 |
| --: | --- | --- |
| 1 | 업무 Level1 | 운영 관리 |
| 2 | 업무 Level2 | 작업자 관리 |
| 3 | 화면 ID | `SFM-WK-001-S-1` |
| 4 | 화면명 | 작업자 상세 |
| 5 | 설명 | 목록에서 선택한 작업자의 계정 정보를 보여주고, 역할과 권한 구역을 수정한다. 저장은 확인 모달을 거친다. |
| 6 | 화면유형 | Sub (목록에서 호출되는 슬라이드 패널) |
| 7 | 개선유형 | 신규 |
| 8 | 작성일 | 2026-06-15 |
| 9 | 작성자 | jisung |

### 화면 레이아웃

```
                                ┌─ 작업자 상세 ──────────────────┐
                                │ 작업자 상세                  ✕ │
                                │ hong@x.com                     │
                                │ 홍길동                          │
                                │ [운영자]                        │
                                ├────────────────────────────────┤
                                │ 계정 상태                       │
                                │ [활성]                          │
                                │                                 │
                                │ 역할 변경                       │
                                │ [ 운영자 ] [ 작업자 ]           │
                                │                                 │
                                │ 권한 구역                       │
                                │ [ 구역 A ] [ 구역 B ]           │
                                ├────────────────────────────────┤
                                │            [ 저장 ]            │
                                └────────────────────────────────┘
```

### 구성

목록에서 작업자를 선택하면 우측에서 슬라이드되어 나오는 패널이다. 헤더에는 이메일, 유저 이름, 현재 역할 칩, 닫기 버튼이 있다. 본문은 계정 상태(활성 또는 비활성) 표시, 역할 변경(운영자 또는 작업자 선택), 권한 구역 토글(구역 A, 구역 B)로 구성된다. 관리자 계정이면 역할 변경 영역에 안내 문구가 표시된다. 변경 사항이 있을 때만 하단 저장 버튼이 활성화되고, 누르면 확인 모달을 거쳐 저장한다.

### 처리절차

1. 목록에서 작업자 선택 (패널 열림)
2. 계정 상태 확인
3. 역할 변경 (운영자 또는 작업자)
4. 권한 구역 토글 (구역 A, 구역 B)
5. 저장 버튼 클릭, 확인 모달에서 확정
6. 닫기

### 제약사항

1. 목록에서 작업자를 선택했을 때만 표시
2. 역할은 운영자와 작업자만 지정 가능 (관리자 계정은 변경 불가, 안내 문구 표시)
3. 변경 사항이 없으면 저장 버튼 비활성
4. 저장은 확인 모달 확정 후 수행
5. API 미연결 시 데모(mock) 데이터로 표시
