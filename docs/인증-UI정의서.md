# UI 정의서: 사용자 인증

사용자 인증은 로그인과 회원가입 두 화면으로 구성한다. 두 화면 모두 사이드바 레이아웃 밖의 단독 화면이며, 왼쪽 브랜드 패널과 오른쪽 입력 폼으로 나뉜 좌우 2단 구성을 공유한다. 화면 폭이 좁으면 왼쪽 브랜드 패널은 숨기고 오른쪽 폼만 보인다.

[로그인 화면]

### 구성

왼쪽은 브랜드 패널로, 어두운 배경 위에 나비 로고 애니메이션과 SKALA 로고, 안내 카피를 배치한다. 이 패널은 넓은 화면에서만 보인다. 오른쪽은 로그인 카드로, 제목과 안내 문구, 이메일과 비밀번호 입력란, 오류 메시지 영역, 아이디 저장 체크박스, 회원가입 이동 링크, 로그인 버튼으로 구성한다. 입력란은 아이콘이 붙은 공통 입력 컴포넌트를 쓴다. 카드 아래에는 저작권 문구를 둔다.

### 처리절차

1. 이메일 입력
2. 비밀번호 입력
3. 로그인 버튼 클릭
4. 성공 시 대시보드로 이동
5. 실패 시 카드 안에 오류 메시지 표시
6. 회원가입 링크로 회원가입 화면 이동

### 제약사항

1. 이메일과 비밀번호는 필수 입력이다
2. 요청 처리 중에는 로그인 버튼을 비활성화하고 진행 문구를 보여준다
3. 로그인 성공 시 토큰과 사용자 정보를 저장하고 이후 요청에 토큰을 첨부한다
4. 아이디 저장은 선택 항목이다

### 구성

왼쪽 브랜드 패널은 로그인 화면과 같은 구성이며 카피 문구만 회원가입 맥락으로 바뀐다. 오른쪽은 회원가입 카드로, 제목과 안내 문구, 이름과 이메일, 비밀번호, 비밀번호 확인 입력란, 오류 메시지 영역, 회원가입 버튼, 로그인 이동 링크로 구성한다.

### 처리절차

1. 이름 입력
2. 이메일 입력
3. 비밀번호 입력
4. 비밀번호 확인 입력
5. 회원가입 버튼 클릭
6. 가입 성공 시 같은 계정으로 자동 로그인하여 대시보드로 이동
7. 실패 시 카드 안에 오류 메시지 표시

### 제약사항

1. 이름과 이메일, 비밀번호는 필수 입력이다
2. 비밀번호와 비밀번호 확인이 일치하지 않으면 오류 메시지를 보여주고 진행하지 않는다
3. 요청 처리 중에는 회원가입 버튼을 비활성화하고 진행 문구를 보여준다
4. 가입 응답에는 토큰이 없으므로 가입 직후 동일 계정으로 로그인을 한 번 더 호출한다
