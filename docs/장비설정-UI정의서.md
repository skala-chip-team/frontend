# UI 정의서 — 장비 설정

> 장비 설정은 ADMIN 전용 기능으로, 사이드바 **'운영자'** 섹션에서 진입한다.
> **① 장비 설정(전체 페이지)**과, 추가 버튼으로 열리는 **② 장비 추가(모달)** 두 화면으로 구성된다. 화면별로 정의를 분리한다.

---

## ① 장비 설정 (전체 페이지)

### 화면 기본 정보

| No. | 항목명 | 내용 |
| --: | --- | --- |
| 1 | 업무 Level1 | 스마트 팩토리 생산 관제 (Smart Factory Monitoring) |
| 2 | 업무 Level2 | 장비 설정 |
| 3 | 화면 ID | `SFM-MC-001-M-0` |
| 4 | 화면명 | 장비 설정 |
| 5 | 설명 | 장비를 목록으로 조회하고, 추가/삭제하며, 각 장비를 특정 공정(STEP)에 매핑한다. 구역·STEP 필터로 탐색한다. (ADMIN 전용) |

### 화면 레이아웃

```
┌──────────────────────────────────────────────────────────────────────┐
│ 장비 설정                                              [ + 장비 추가 ] │ ← 헤더 + 추가 버튼
├──────────────────────────────────────────────────────────────────────┤
│ 구역   [ 전체 ][ 구역 A ][ 구역 B ]                            총 10대 │ ← 구역 필터 + 총 대수
│ STEP   [ 전체 ][ STEP A ][ STEP B ][ STEP C ][ STEP D ]               │ ← STEP 필터(구역 아래)
├──────────────────────────────────────────────────────────────────────┤
│ 장비 ID     타입     구역    담당 STEP   상태       관리                │ ← 테이블 헤더
│ ───────────────────────────────────────────────────────────────────── │
│ MACHINE-01  TYPE_A   구역 A   STEP A     [가동]      [삭제]            │
│ MACHINE-02  TYPE_A   구역 A   STEP A     [점검중]    [삭제]            │
│ MACHINE-10  TYPE_B   구역 A   STEP B     [대기]      [삭제]            │
│ MACHINE-22  TYPE_C   구역 A   STEP C     [정지]      [삭제]            │
│        …                                                              │
└──────────────────────────────────────────────────────────────────────┘
```

### 구성

조회·관리 중심 화면으로, 상단 헤더/필터 영역과 하단 테이블 영역으로 구분된다. 헤더에는 화면 제목과 우측 `[+ 장비 추가]` 버튼이 있다. 필터 영역은 **구역 필터(전체/구역 A/구역 B)** 와 그 아래 **STEP 필터(전체/STEP A~D)** 가 세로로 배치되며, 구역 줄 우측에 총 대수를 표기한다. 테이블은 장비 ID·타입·구역·담당 STEP·상태(색상 Chip)·관리(삭제) 컬럼으로 구성되고, 각 행의 [삭제]로 해당 장비를 제거한다. (수정 기능은 제공하지 않는다.)

### 처리절차

1. 구역 필터 선택 (전체 / 구역 A·B)
2. STEP 필터 선택 (전체 / STEP A~D)
3. [+ 장비 추가] → ② 장비 추가 모달
4. 행의 [삭제] → 삭제 확인 모달 → 확인 시 제거

### 제약사항

- ADMIN 권한 전용 (사이드바 '운영자' 섹션, `/machines`)
- 상태 Chip 색: 가동=초록 / 점검중=주황 / 정지=빨강 / 대기=회색
- 삭제는 확인 모달 후 수행(되돌릴 수 없음). 진행 중 스케줄이 있는 장비는 서버가 거절할 수 있음(메시지 노출)
- API 미연결 시 mock(세션 메모리)으로 동작 — 추가/삭제는 세션 동안 유지, 새로고침 시 초기화

## ② 장비 추가 (모달)

### 화면 기본 정보

| No. | 항목명 | 내용 |
| --: | --- | --- |
| 1 | 업무 Level1 | 스마트 팩토리 생산 관제 (Smart Factory Monitoring) |
| 2 | 업무 Level2 | 장비 설정 |
| 3 | 화면 ID | `SFM-MC-001-P-1` |
| 4 | 화면명 | 장비 추가 |
| 5 | 설명 | 새 장비를 등록한다. 장비 타입·구역·담당 STEP을 지정하며, 장비 ID는 저장 시 자동 생성되고 상태는 '대기'로 시작한다. |
| 6 | 화면유형 | Popup (목록 화면에서 호출되는 모달) |
| 7 | 개선유형 | 신규 |
| 8 | 작성일 | 2026-06-15 |
| 9 | 작성자 | jisung |

### 화면 레이아웃

```
        ┌─ 장비 추가 ───────────────────────────────┐
        │                                       ✕  │ ← 닫기
        │ 장비 타입 [ TYPE_A ▾ ]   구역 [ 구역 A ▾ ] │ ← 타입 · 구역
        │ 담당 STEP [ STEP A ▾ ]                     │ ← 담당 STEP(구역 종속)
        │                                           │
        │                        [ 취소 ]  [ 추가 ]  │
        │       장비 ID는 저장 시 자동 생성됩니다.      │ ← 하단 작은 안내 문구
        └───────────────────────────────────────────┘
```

### 구성

가운데 모달 폼으로, 입력 항목과 액션·안내로 구성된다. 입력 항목은 **장비 타입(TYPE_A~D)**, **구역(구역 A/B)**, **담당 STEP(STEP A~D)** 세 가지다. STEP은 선택한 구역에 속한 것만 노출되며, 구역을 바꾸면 STEP이 해당 구역의 첫 STEP으로 자동 보정된다. 하단에는 취소/추가 버튼과, 그 아래 "장비 ID는 저장 시 자동 생성됩니다." 안내 문구가 작게 표시된다. 상태는 입력받지 않고 추가 시 '대기'로 고정된다.

### 처리절차

1. 장비 타입 선택
2. 구역 선택 (→ 담당 STEP 옵션 갱신)
3. 담당 STEP 선택
4. [추가] → 장비 ID 자동 생성·상태 '대기'로 등록 → 모달 닫힘, 목록 갱신
5. [취소]/✕ → 입력 취소

### 제약사항

- 필수: 장비 타입·구역·담당 STEP (모두 선택해야 [추가] 활성)
- 장비 ID는 서버(또는 mock)가 자동 생성(MACHINE-NN), 사용자 입력 불가
- 상태는 추가 시 '대기' 고정 (선택 불가)
- 장비-공정 매핑은 1:1 (장비 1대 = STEP 1개)
- API 미연결 시 mock으로 등록(세션 유지)
