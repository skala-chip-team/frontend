# 커밋 컨벤션

## 형식
```
[타입]: [내용]
```
- 메시지는 **한글**로 작성
- 타입은 영문 PascalCase

## 타입
| 타입 | 설명 |
|------|------|
| `Feature` | 새로운 기능 추가 |
| `Fix` | 버그 수정 |
| `Docs` | 문서 수정 |
| `Style` | 코드 포맷팅, 세미콜론 누락 등 코드 변경 없는 경우 |
| `Refactor` | 코드 리팩토링 |
| `Test` | 테스트 코드 추가 및 리팩토링 |
| `Chore` | 빌드 업무, 패키지 매니저 설정, 프로젝트 관리 관련 |
| `Design` | CSS 등 사용자 UI 디자인 변경 |
| `Comment` | 주석 추가 및 변경 |
| `Rename` | 파일 또는 폴더 이름 수정/이동 |
| `Remove` | 파일 삭제 |
| `!BREAKING CHANGE` | 커다란 API 변경 |
| `!HOTFIX` | 치명적인 버그 긴급 수정 |

## 커밋 단위 규칙
- 파일 하나, 스텝 하나 = 커밋 X
- **의미 있는 기능/작업 영역 단위**로 묶을 것
- 기준: "이 커밋 하나로 무엇이 동작하는가?" 설명 가능해야 함

## 예시
```
Feature: 로그인 페이지 구현
Fix: 토큰 만료 시 리다이렉트 오류 수정
Chore: 프로젝트 초기화 및 개발 환경 설정
Design: 헤더 컴포넌트 반응형 레이아웃 적용
Refactor: API 레이어 axios 인스턴스 분리
```
